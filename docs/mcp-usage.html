MCP Tool Usage Guide

Common Error

If you see this error: MCP error -32602: Invalid arguments for tool get_filesystem_metadata: [{"code":"invalid_type","expected":"string","received":"undefined","path":["path"],"message":"Required"}]

It means you're not providing the required path parameter when calling the get_filesystem_metadata tool.

Correct Usage

When using the MCP client, you must always provide all required parameters for each tool:

1. Get Filesystem Metadata

mcp-client call-tool get_filesystem_metadata '{"path": "./package.json"}'

The path parameter is required!

2. Get Weather Forecast

mcp-client call-tool get_weather_forecast '{"location": "New York"}'

The location parameter is required!

3. Get System Info

mcp-client call-tool get_system_info '{}'

This tool doesn't require any parameters.

4. Verify OAuth Token

mcp-client call-tool verify_oauth_token '{"token": "YOUR_TOKEN", "service": "github"}'

Both token and service parameters are required!

Running the MCP Server

To start the MCP server:

npm start

The server uses STDIO transport by default.
